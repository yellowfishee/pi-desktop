## 1. Tool results 双重渲染 — 已修复

**根因**: `finalizeToolCall()` 在 `message_update(toolcall_end)` 事件时执行，它用 spread 覆盖了整个 ContentBlock，导致：
- `toolStatus` 被重置为 `'running'`（覆盖了 `tool_execution_end` 已设的 `'success'`/`'error'`）
- `toolResult` 被丢失（spread 没有 `...b` 保留已有字段）

**时序**: pi 的事件流中 `tool_execution_end` 可能早于或晚于 `toolcall_end`，两种情况都会出问题。

**修复**: `finalizeToolCall` 中保留已有的 `toolResult` 和 `toolStatus`：
```js
toolResult: b.toolResult,
toolStatus: b.toolResult ? b.toolStatus : ('running' as const),
```

## 2. TPS/extension status 重定位 — 已完成

ExtensionStatusBar 已在 ChatPanel 中 MessageInput 下方、StatusBar 上方，且仅在 isStreaming 时显示。StatusBar 中已无 TPS/extension status 残留。
