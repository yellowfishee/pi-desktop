# Step 3.4: 进程诊断页面

## 目标
提供 pi 进程的健康诊断页面，展示进程状态、心跳历史、崩溃恢复日志。

## 实施链路

### 1. 诊断数据收集
- 在 Rust 后端添加 `get_pi_diagnostics` 命令
- 返回：pi 进程 PID、启动时间、运行时长、心跳次数、崩溃次数、最后错误
- 前端通过 `invoke('get_pi_diagnostics')` 获取

### 2. 诊断 UI
- 新建 `DiagnosticsPanel` 组件（或嵌入 SettingsPanel 新 tab）
- 显示：
  - pi 进程状态（运行中/已停止） + 绿色/红色指示灯
  - 进程 PID
  - 启动时间 + 运行时长
  - 心跳检测历史（最近 10 次）
  - 崩溃恢复次数 + 最近一次崩溃时间 + 错误信息
  - 重启按钮

### 3. 设置面板集成
- Settings 新增"诊断"tab
- 或在 StatusBar 中添加快捷入口（⚙ 图标）

## 验收标准
- [ ] 诊断页面可查看 pi 进程状态
- [ ] 显示 PID、运行时长、心跳历史
- [ ] 显示崩溃恢复记录
- [ ] 可手动重启 pi 进程
