# Step 3.1: 消息编辑重发

## 目标
双用户消息进入编辑模式，修改后通过 fork 重新发送，创建分支会话。

## 实施链路

### 1. 双击编辑
- 用户消息气泡响应 `onDoubleClick`
- 进入编辑模式：显示 textarea 替代原有文本，图片保留
- 显示"取消"和"发送"按钮

### 2. Fork 发送
- 编辑模式下点击"发送"
- 调用 `sendCommand({ type: 'fork', entryId: message.entryId })`
- pi 返回分支会话的 sessionId + sessionFile
- 切换会话，在 fork 的新分支中发送编辑后的文本作为 prompt

### 3. get_fork_messages 支持
- 在编辑前调用 `sendCommand({ type: 'get_fork_messages' })`
- 获取具有 entryId 的可 fork 消息列表
- 渲染时标记哪些消息可编辑（有 entryId 的显示编辑光标）

### 4. entryId 来源
- `agent_end` 事件中可能包含 entryId
- 或从 `get_messages` / `get_state` 响应中提取
- 已有的 UIMessage.entryId 字段应该从后端响应中填充

## 验收标准
- [ ] 双击用户消息进入编辑模式
- [ ] 编辑后点击发送，通过 fork 创建新分支
- [ ] 编辑完成后自动切换到新分支会话
- [ ] 无 entryId 的消息不显示编辑提示
- [ ] 取消按钮退出编辑模式
