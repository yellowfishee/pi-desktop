# 项目功能完整修复计划

## 目标

修复当前已确认的会话、路径、进程、多窗口、诊断、Git、配置、流式显示和首次启动问题，使界面状态与 Pi 实际状态一致，并为关键行为增加自动化测试。

## 已确认方案

- 每个 Tauri 窗口使用独立 Pi 子进程、命令队列、诊断状态和事件通道。
- Git 和项目操作使用会话头中的真实 `cwd`，不再从目录编码反推项目路径。
- 前端测试使用 Vitest 和 React Testing Library。
- 创建 `feature/project-functional-repairs` 分支。
- 每完成一个步骤提交一次，提交后等待用户审核。
- 不增加旧接口兼容层、回退接口或废弃路径。

## 范围

1. 建立测试基础和明确的前后端类型边界。
2. 重构 Pi 进程为窗口级实例，修复 stderr、退出清理、命令泄漏、自动重启和诊断。
3. 修复项目真实路径、会话扫描、路径边界和安全文件写入。
4. 修复分支会话加载、会话名称、置顶和会话切换一致性。
5. 完成 Git 变更操作界面、新仓库支持和刷新机制。
6. 完成配置、窗口状态、上次会话和首次启动体验。
7. 修复流式显示、Markdown 原文、通知生命周期和界面过期状态。
8. 完成全量验证并同步当前功能说明。

## 不在范围内

- 不改变 Pi RPC 协议。
- 不引入旧 Tauri 命令参数兼容层。
- 不新增与已确认问题无关的功能。
- 不修改 `AGENTS.md`、`.gitignore` 或 `.env`。

## 步骤链路

1. `step/01-test-contracts.md`：测试基础与接口契约。
2. `step/02-window-process-runtime.md`：窗口级 Pi 运行时与诊断。
3. `step/03-session-project-safety.md`：真实项目路径与会话文件安全。
4. `step/04-session-consistency.md`：分支会话和元数据一致性。
5. `step/05-git-workflow.md`：完整 Git 工作流。
6. `step/06-persistence-onboarding.md`：配置持久化和首次启动。
7. `step/07-rendering-notifications.md`：流式渲染、Markdown 和通知。
8. `step/08-final-verification.md`：全量验收与说明同步。
