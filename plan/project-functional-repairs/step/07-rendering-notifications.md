# 步骤 7：流式渲染、Markdown 和通知

## 目标

保证持续流式输出可见、展示文本不被擅自改写，并清理已经结束的状态通知。

## 实施链路

1. 删除基于反复重置定时器的流式文本延迟。
2. 直接消费已经通过动画帧批处理的 Store 更新，限制 Markdown 重渲染成本而不延迟文本可见性。
3. 删除 `normalizeAwkwardLineBreaks`，按模型原文渲染和复制。
4. 为代码复制失败提供错误提示，并清理延迟计时器。
5. 为自动重试、自动重启和压缩通知增加稳定标识。
6. 状态结束时更新或删除对应通知，不累积永久“进行中”通知。
7. 清理事件监听器、动画帧和定时器，避免窗口卸载后更新状态。
8. 消除会吞掉用户操作错误的空 `catch`，向界面显示可执行的错误信息。

## 验收标准

- 高频连续 token 输出持续可见，不在结束时一次性跳变。
- 中文换行、诗歌、日志和手工排版保持原样。
- 自动重试结束后没有残留“重试中”通知。
- 相同后台任务只显示一条可更新通知。
- 窗口关闭后没有挂起的动画帧或状态更新。
- 复制失败和关键操作失败会向用户说明原因。
