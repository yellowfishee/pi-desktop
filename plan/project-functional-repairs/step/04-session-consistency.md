# 步骤 4：会话分支与元数据一致性

## 目标

确保桌面端显示的消息、名称、置顶状态和当前会话与 Pi 实际状态一致。

## 实施链路

1. 切换会话时先调用 `switch_session`，确认未被扩展取消。
2. 切换成功后调用 Pi RPC `get_messages`，以 Pi 返回的当前分支消息替换界面消息。
3. 获取 `get_state` 后同步模型、思考级别、会话 ID、文件和消息统计。
4. 切换失败或取消时恢复原会话和原消息，不提前清空。
5. 删除直接读取全部 JSONL 消息的界面路径。
6. 扫描完整会话文件，选择最新的 `session_info` 名称和置顶值。
7. 新增 `session_info` 时追加合法树条目，保留第 1 行会话头，并使用当前叶节点作为 `parentId`。
8. 当前会话重命名优先使用 Pi RPC `set_session_name`；非当前会话通过安全文件更新完成。

## 验收标准

- 含废弃分支的会话只显示当前分支。
- 会话切换被取消时界面不丢失现有消息。
- 多次重命名显示最后一次名称。
- 长会话第 200 行后的名称和置顶信息仍能读取。
- 置顶未命名会话不破坏首行会话头。
- Pi 重启后恢复的会话和界面会话一致。
