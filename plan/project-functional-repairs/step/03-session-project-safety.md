# 步骤 3：真实项目路径与会话文件安全

## 目标

删除有歧义的项目路径反解逻辑，并限制所有会话文件操作只能作用于 Pi 会话目录。

## 接口设计

- `ProjectMeta.path` 来自会话头的真实 `cwd`。
- Git 命令接收 `projectPath`，不再接收 `dirName`。
- 删除项目仍使用扫描得到的会话目录标识，但后端必须规范化并验证边界。
- 会话读取、修改和删除接收会话路径，但后端必须验证路径位于配置的会话根目录且扩展名为 `.jsonl`。

## 实施链路

1. 扫描会话时读取首行 `session` 头中的 `cwd` 和会话 ID。
2. 按真实 `cwd` 分组项目，不从目录编码推导路径。
3. 删除 `decode_project_path` 及其调用。
4. 增加会话根目录解析和路径规范化函数。
5. 拒绝绝对越界、`..` 越界、符号链接越界、非 JSONL 文件和非目录目标。
6. 会话元数据修改使用同目录临时文件、flush、同步和原子替换。
7. 删除会话和项目继续要求前端确认，后端只执行通过边界校验的目标。

## 验收标准

- `pet-platform` 等包含连字符的项目路径保持不变。
- Windows、macOS 和 Linux 路径都由会话头直接取得。
- 无法读取头信息的文件不会产生伪造项目路径。
- 任意会话目录外路径都会被拒绝。
- 写入失败时原会话文件保持完整。
- 不添加旧 `dirName` Git 参数兼容处理。
