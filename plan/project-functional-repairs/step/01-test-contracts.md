# 步骤 1：测试基础与接口契约

## 目标

建立可重复执行的前端和 Rust 回归测试，先固定正确行为，再开始结构性修改。

## 实施链路

1. 检查现有依赖，确认项目没有测试框架。
2. 添加 Vitest、jsdom、React Testing Library 和必要的类型依赖。
3. 增加测试脚本和 Vitest 配置。
4. 为 RPC 响应、项目元数据、窗口标识、Git 状态和配置建立明确 TypeScript 类型。
5. 为会话路径校验、会话元数据读取、Git 状态解析和进程状态转换建立 Rust 单元测试入口。
6. 将后续步骤需要验证的行为写为失败测试或测试夹具。

## 测试范围

- 分支会话只展示当前分支。
- 项目路径保留目录名中的连字符。
- 会话元数据更新不改变文件头。
- 不允许会话目录外的读取、修改或删除。
- 新 Git 仓库没有 `HEAD` 时仍能列出变更。
- 多窗口命令和事件按窗口隔离。
- 自动重启次数在稳定运行前不会重置。
- 配置往返保存不丢字段。
- 流式文本持续更新。
- 自动重试通知在结束后消失。

## 验收标准

- `pnpm test --run` 可以执行。
- `cargo test --manifest-path src-tauri/Cargo.toml` 可以执行。
- 测试夹具不读写真实用户会话目录。
- TypeScript 业务代码不再通过新增 `any` 绕过新接口。
- 测试能在无 Tauri 窗口和无 Pi 子进程的环境运行。
