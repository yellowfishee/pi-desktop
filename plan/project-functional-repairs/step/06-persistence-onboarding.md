# 步骤 6：状态持久化与首次启动

## 目标

完整保存并恢复窗口、面板、外观、Pi 路径和上次会话，并向首次使用或环境缺失的用户展示明确引导。

## 配置设计

配置包含：

- 各窗口的位置、大小和最大化状态。
- 侧边栏、属性面板和 Changes 面板的开关与宽度。
- 主题、字体和字号。
- Pi 路径。
- 上次项目和会话文件。

多窗口配置按窗口标签保存独立窗口状态，共享外观和 Pi 路径。

## 实施链路

1. 扩展 Rust 配置结构并增加输入范围验证。
2. 监听 Tauri 窗口移动、缩放、最大化和关闭事件。
3. 防抖保存真实状态，应用退出前强制 flush。
4. 启动时先恢复布局，再启动窗口 Pi 运行时。
5. 上次会话存在且通过路径校验时恢复会话。
6. 接入 `WelcomeScreen`，覆盖 Pi 缺失、Bash 缺失、路径无效和重试流程。
7. 环境就绪后进入主界面；设置面板仍可随时修改环境。

## 验收标准

- 重启后窗口位置、尺寸、最大化状态和面板宽度一致。
- 多个窗口分别恢复各自布局。
- 主题、字体、字号和 Pi 路径不会因调整其他设置而丢失。
- 上次会话不存在时显示明确状态，不自动选择错误会话。
- 首次启动时引导页可以完成检测、选择路径和重试。
