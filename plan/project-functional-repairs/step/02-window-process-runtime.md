# 步骤 2：窗口级 Pi 运行时与诊断

## 目标

让每个桌面窗口拥有独立 Pi 子进程，并确保进程输入输出、退出、重启和诊断状态真实可靠。

## 接口设计

- 用窗口标签作为运行时实例键。
- `start_pi`、`stop_pi`、`pi_is_running`、`send_command`、`send_extension_ui_response` 和 `get_pi_diagnostics` 从调用窗口解析实例。
- Pi 事件只发送给所属窗口，不再广播给全部窗口。
- 窗口关闭时停止该窗口的 Pi 子进程并清理待处理命令。

## 实施链路

1. 将单份 `AppState` 进程字段替换为窗口运行时注册表。
2. 为每个窗口创建 stdin、child、待处理命令、诊断和生命周期状态。
3. 同时消费 stdout 和 stderr；stderr 进入诊断缓冲并避免管道阻塞。
4. 统一处理 EOF、读取错误、进程退出和主动停止。
5. 所有序列化、写入和 flush 失败路径清理对应待处理命令。
6. 记录实际 PID、开始时间、运行时长、退出码、崩溃次数和最近错误。
7. 用明确的健康窗口重置自动重试计数，不在刚启动成功时重置。
8. 取消窗口卸载后的延迟重启任务。

## 验收标准

- 两个窗口可以同时使用不同会话，命令和事件不串线。
- 一个窗口重启或关闭 Pi 不影响其他窗口。
- 大量 stderr 输出不会阻塞 RPC。
- 异常退出后 `running`、PID 和崩溃信息立即更新。
- 写入失败、超时和退出后，待处理命令表没有残留。
- 连续快速崩溃达到上限后停止重启。
- 主动停止不会触发自动重启。
