# Step 5.5: 消息反馈 + 滚动优化

## 目标
添加消息点赞/踩反馈机制，优化自动滚动和回到底部按钮体验。

## 实施链路

### 1. 消息反馈按钮
- 在 AssistantBubble 底部添加 👍 / 👎 按钮
- 点赞/踩时短暂动画反馈
- 数据存储在本地（无需 pi 后端支持），后续可接入 pi feedback API
- 踩时可选弹出原因选择

### 2. 重新生成
- 在 AssistantBubble 右键或底部添加"重新生成"按钮
- 实现：fork 当前消息 → 在新分支重新发送相同的 prompt
- 或发送 abort + 重新 prompt

### 3. 回到底部按钮优化
- 当前按钮在输入框上方，增强可见性
- 点击时添加平滑动画
- 有新消息生成时按钮自动隐藏
- 添加未读消息计数气泡

## 验收标准
- [ ] 点赞/踩按钮可见且可用
- [ ] 重新生成功能正常
- [ ] 回到底部按钮体验流畅
