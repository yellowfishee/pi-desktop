Step 1：Git Changes 面板增强

Rust + React

将 ChangesPanel 从「只读 diff 查看器」升级为「交互式 Git 操作面板」。支持 staging/unstaging/discard/commit 完整工作流。

1.1 Rust 后端 → 1.2 UI 操作 → 1.3 Commit 对话框

Step 1.1：Rust 后端 git 操作命令

在 commands.rs 中新增 4 个 Tauri Command：

  • stage_files(dirName, files) → git add
  • unstage_files(dirName, files) → git reset HEAD --
  • discard_changes(dirName, files, staged) → git checkout --
  • git_commit(dirName, message, files?) → git commit -m

注册到 generate_handler![]。前端 tauri.ts 新增对应的 invoke 封装函数。

Step 1.2：ChangesPanel 文件操作 UI

对 ChangesPanel.tsx 进行以下增强：

  • 每个文件行添加 checkbox，支持多选
  • 顶部工具栏：全选 · Stage · Unstage · Discard · Commit
  • 根据文件状态（?? / M / A / D）显示不同的可操作按钮
  • 操作后自动调用 listGitChanges 刷新

Step 1.3：Commit 对话框

新增 CommitDialog.tsx 组件：

  • Modal 遮罩层（ESC 关闭）
  • commit message 文本输入区
  • 显示将提交的文件列表
  • 提交后 toast 提示 + 自动刷新变更列表
依赖关系：1.1 → 1.2 → 1.3 线性依赖。
不依赖 Step 2 或 Step 3，可独立推进。

验收标准

  • cargo check 编译通过，4 个 command 注册到 handler
  • 文件列表 checkbox 选中/全选功能正常
  • Stage/Unstage/Discard 操作按文件状态正确显示并执行
  • 操作后自动刷新变更列表
  • Commit 对话框 UI + 输入 + loading + 成功/失败 toast
  • 错误提示 toast

Step 2：代码高亮 & Markdown 增强

React + CSS

修复当前 Markdown 渲染的体验缺陷：暗色模式代码块不可读、diff 无高亮、缺少行号。

2.1 暗色主题高亮 | 2.2 Diff 语法高亮 | 2.3 行号 + 复制反馈

Step 2.1：暗色主题高亮 + 动态切换

当前固定导入 highlight.js/styles/github.css（亮色），暗色模式下不可读。

方案：在 styles.css 中用 CSS 自定义属性 + .dark 选择器覆盖 hljs 样式。

.hljs { color: #24292e; background: #f6f8fa; }
.dark .hljs { color: #c9d1d9; background: #0d1117; }
/* ... token 颜色映射 */

Step 2.2：Diff 语法高亮

确保 highlight.js 内置的 diff 语言被正确识别，在 styles.css 中增强表现：

  • .hljs-addition → 绿色背景（新增行）
  • .hljs-deletion → 红色背景（删除行）
  • .hljs-meta → 蓝色（@@、diff --git 等）

Step 2.3：行号 + 复制按钮状态反馈

代码块渲染时拆分行为独立 DOM 节点，左侧显示灰色行号（user-select: none）。

复制按钮增加 useState 管理「已复制」状态反馈（图标变化 + 文字，2s 恢复）。

验收标准

  • 暗色模式下代码块可读
  • 亮色/暗色切换时 hljs 颜色跟随
  • ```diff 代码块有 +/- 颜色区分
  • 代码块左侧有灰色行号（不影响选择/复制）
  • 复制按钮点击后有「已复制」状态反馈

Step 3：虚拟滚动

React

当前消息列表一次渲染所有 DOM，长会话性能差。使用已安装的 @tanstack/react-virtual ^3.14.2 实现虚拟滚动。

3.1 react-virtual 集成 → 3.2 自动滚动适配

Step 3.1：集成 @tanstack/react-virtual

在 ChatPanel.tsx 中使用 useVirtualizer：

  • 用 estimateSize: 200 估算高度 + measureElement 回调动态修正
  • overscan: 5 预渲染上下各 5 条
  • 保留空消息状态和 loading 状态
const virtualizer = useVirtualizer({
  count: messages.length,
  getScrollElement: () => scrollRef.current,
  estimateSize: () => 200,
  overscan: 5,
});

const virtualItems = virtualizer.getVirtualItems();

Step 3.2：自动滚动行为适配

从 scrollIntoView() 迁移到 virtualizer.scrollToIndex()：

  • 流式输出自动滚动：virtualizer.scrollToIndex(lastIndex, { align: 'end' })
  • 用户上滚暂停自动滚动（使用 virtualizer.scrollOffset 判断）
  • 「回到底部」按钮适配虚拟滚动

验收标准

  • DOM 中仅渲染可视 + overscan 区域
  • 不同高度消息气泡（含展开工具卡片）正常渲染
  • 流式输出时自动滚动跟随最新内容
  • 用户手动上滚后暂停自动滚动
  • 「回到底部」按钮功能正常
  • 空消息 / loading 状态正常工作
