# Step 2.5: Extension UI 对话框

## 目标
完整实现 extension_ui_request 协议，让 pi 扩展可以通过 GUI 与用户交互。

## 实施链路

### 1. Extension UI 请求监听
- 已在 tauri.ts 中监听 `pi-extension-ui-request` 事件
- 需要创建 Modal 组件处理各种 method

### 2. Dialog 方法（需要响应）
- `select` → 下拉选择对话框
- `confirm` → 确认对话框
- `input` → 文本输入对话框
- `editor` → 多行文本编辑器对话框

### 3. Fire-and-forget 方法（仅展示）
- `notify` → Toast 通知（已有 NotificationStack）
- `setStatus` → 状态栏更新（已有 extensionStatuses）
- `setWidget` → Widget 展示区
- `setTitle` → 窗口标题更新
- `set_editor_text` → 输入框预填充

### 4. 响应发送
- 调用 `invoke('send_extension_ui_response', { response })` 回复
- 响应格式：`{ type: 'extension_ui_response', id, value/confirmed/cancelled }`

### 5. ExtensionDialogModal 组件
- 通用 Modal 框架
- 根据 method 渲染不同内容
- 支持 timeout 倒计时显示
- 取消按钮发送 cancelled 响应

## 验收标准
- [ ] select 对话框正确展示选项列表
- [ ] confirm 对话框正确展示确认提示
- [ ] input 对话框正确展示文本输入
- [ ] editor 对话框正确展示多行编辑器
- [ ] 所有对话框的响应正确回传 pi
- [ ] 取消按钮发送 cancelled 响应
- [ ] notify 转为 Toast 通知
- [ ] setStatus 更新状态栏
