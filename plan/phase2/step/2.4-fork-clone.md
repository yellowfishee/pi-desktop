# Step 2.4: Fork / Clone + 消息分支

## 目标
支持从历史消息处分叉对话，创建分支会话。

## 实施链路

### 1. 消息操作菜单
- 用户消息气泡悬停显示 "..." 菜单按钮
- 菜单选项：Fork from here / Clone session

### 2. Fork
- 调用 `sendCommand({ type: 'fork', entryId })`
- 需要先获取 entryId（从消息元数据中）
- 返回 fork 的原始文本，创建新分支

### 3. Clone
- 调用 `sendCommand({ type: 'clone' })`
- 复制当前分支为新会话

### 4. get_fork_messages
- 调用 `sendCommand({ type: 'get_fork_messages' })`
- 获取可用于 fork 的用户消息列表
- 在 fork 前展示可选消息列表

### 5. 消息元数据
- UIMessage 需要添加 `entryId` 字段
- 从 agent_end 消息或 get_messages 响应中获取

## 验收标准
- [ ] 用户消息可显示操作菜单
- [ ] Fork 从指定消息创建分支
- [ ] Clone 复制当前会话
- [ ] Fork 后自动切换到新分支
