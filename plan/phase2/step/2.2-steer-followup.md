# Step 2.2: Steer / FollowUp 队列交互

## 目标
当 pi 正在生成时，用户输入的消息应该作为 steer 或 followUp 排队，而不是被忽略。

## 实施链路

### 1. MessageInput 发送逻辑
- 当 `isStreaming === true` 时，发送改为 `streamingBehavior: 'steer'`
- 或者添加 UI 让用户选择 steer / followUp
- 最简方案：流式中直接走 steer，输入框不禁用

### 2. 队列状态展示
- 监听 `queue_update` 事件，更新 `steeringQueue` / `followUpQueue`
- 在输入框下方或状态栏显示排队消息数量
- 已有 `useUIStore.steeringQueue / followUpQueue`

### 3. 输入框行为
- 流式中输入框不禁用（`disabled={isStreaming}` 改为可输入）
- placeholder 改为 "输入补充指令..."
- 发送时用 `prompt` + `streamingBehavior: 'steer'`

## 验收标准
- [ ] 流式中输入框可输入
- [ ] 发送的消息作为 steer 排队
- [ ] queue_update 事件正确更新队列状态
- [ ] 排队消息数在 UI 中可见
